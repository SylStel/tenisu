
Améliorations à apporter à mon travail :

Les tests unitaires :
- je n'ai pas codé tous les cas de figures, ni toutes les erreurs possible par manque de temps.
J'ai laissé du code commenté notamment pour les tests des fonctions du Service.
Dans l'idéal j'aurais ajouté des builder aux classes TennisPlayer et DaoTennisPlayer pour pouvoir créer ces pojos
facilement et ainsi pouvoir tester tous les cas de figure plus facilement.
-l'ajout de tests d'intégration seraient aussi une bonne garantie de qualité du code dans le temps.

-il faut aussi rajouter des logger dans les classes pour pouvoir comprendre chaque étape des fonctions.
surtout pour le débug et les erreurs.

-j'aurais pu afficher les images en rajoutant une balise html dans la string renvoyée par mes fonctions.

-Dans ce projet je n'ai fait qu'une seule classe Service. Il serait mieux de la séparer en 3 classes pour
distinguer les fonctions générant les statistiques, l'affichage de la liste des joueurs et l'affichage des info
d'un joueur. Et de rajouter une nouvelle classe pour chaque nouvelle fonctionnalité.

J'ai codé mon programme sous intellIJ et je l'ai testé en local à l'aide de Docker.
J'ai voulu mettre sur le cloud via AWS, mais je manque de connaissances pour maîtriser ce système complexe en quelques heures.
Si vous avez un moyen simple à me recommander pour mettre le programme sur le cloud je pourrai le faire rapidement.
